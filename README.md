# 키워드 노출량 대시보드

네이버와 인스타그램의 키워드별 노출량을 시각화하는 인터랙티브 웹 대시보드입니다.

## 📊 프로젝트 개요

매일 수집되는 지점별 키워드 노출량 데이터를 구글 스프레드시트에서 자동으로 불러와, 두 가지 형태의 대시보드로 시각화합니다.

### 주요 기능

#### 🗓️ 대시보드 1: 일별 키워드 노출량
- 날짜 선택 드롭다운으로 특정 날짜의 데이터 조회
- 지점별로 카드 형태로 구분된 레이아웃
- 플랫폼별(네이버/인스타그램) 구분 표시
- 메인 키워드와 카테고리 키워드의 노출량을 테이블로 표시
- 네이버: 메인 키워드 + 카테고리 키워드 1~3 및 각각의 포스팅 수
- 인스타그램: 메인 키워드 + 총 노출 포스팅 수

#### 📈 대시보드 2: 전주 대비 증감률 분석
- 지점별로 그룹화된 비교 분석
- 플랫폼별(네이버/인스타그램) 구분
- 최근 7일 평균 vs 이전 7일 평균 비교
- 증감률을 퍼센트로 표시 (증가: 초록색 ↑ / 감소: 빨간색 ↓)
- Chart.js를 활용한 막대 그래프 시각화
- 키워드별 상세 수치 제공

### 데이터 구조

스프레드시트는 다음과 같은 컬럼 구조를 가집니다:

| 컬럼 | 설명 |
|------|------|
| 날짜 (B열) | MM/DD 형식 (예: 02/04) |
| 매장명 (C열) | 강남점, 명동점, 대학로점, 건대점, 천안점, 안산점, 구월점 |
| 플랫폼 (D열) | 네이버, 인스타그램 |
| main 키워드 (E열) | 메인 검색 키워드 |
| 총 노출 포스팅 수 (F열) | 해당 키워드의 총 노출 수 |
| 카테고리 키워드 1~3 (G, I, K열) | 관련 카테고리 키워드 |
| 키워드 1~3 포스팅 수 (H, J, L열) | 각 카테고리 키워드의 포스팅 수 |

### 사용 방법

#### 방법 1: 독립 실행 (젠슨파크 없이) ⭐
1. **파일 열기**: `standalone.html` 파일을 Chrome 브라우저로 열기
2. **데이터 자동 로드**: 자동으로 구글 시트에서 데이터 가져옴
3. **대시보드 사용**: 날짜 선택 및 탭 전환으로 데이터 분석

#### 방법 2: 웹 호스팅 (URL 공유용)
1. **GitHub Pages**: 무료로 웹사이트 배포 (권장)
2. **Google Apps Script**: 구글 시트와 직접 연동
3. **상세 가이드**: `DEPLOYMENT.md` 참고

#### 기본 사용법
1. **대시보드 전환**: 상단 탭을 클릭하여 "일별 노출량"과 "전주 대비 증감" 대시보드를 전환
2. **날짜 선택**: 일별 대시보드에서 드롭다운으로 날짜 선택
3. **데이터 새로고침**: 우측 상단 "새로고침" 버튼으로 최신 데이터 업데이트

## 🛠️ 기술 스택

## 🎨 주요 컴포넌트

### 1. Data Loader (data-loader.js)
- Google Sheets에서 CSV 형식으로 데이터 fetch
- CSV 파싱 및 구조화
- 날짜별, 지점별 데이터 그룹화
- 전주 대비 증감률 계산 로직

### 2. Daily Dashboard (dashboard-daily.js)
- 날짜 선택 기능
- 지점별 카드 렌더링
- 플랫폼별(네이버/인스타그램) 데이터 표시
- 키워드 테이블 생성

### 3. Comparison Dashboard (dashboard-comparison.js)
- 주간 데이터 집계 및 비교
- 증감률 계산 및 시각화
- Chart.js를 활용한 막대 그래프
- 컬러 코딩된 증감 표시 (증가: 초록, 감소: 빨강)

### 4. Main Controller (main.js)
- 애플리케이션 초기화
- 탭 전환 관리
- 데이터 새로고침
- 로딩/에러 상태 관리
- Toast 알림 시스템

## 📱 반응형 디자인

## 🔄 데이터 업데이트 방법

1. 구글 스프레드시트에서 새로운 데이터 추가
2. 대시보드에서 "새로고침" 버튼 클릭
3. 자동으로 최신 데이터가 반영됨

## ⚠️ 주의사항

### Google Sheets 권한 설정 ⚠️
- **필수**: 스프레드시트의 공유 설정이 "링크가 있는 모든 사용자"로 설정되어야 합니다
- CSV Export URL이 접근 가능해야 합니다
- 설정 방법:
  1. 구글 시트 우측 상단 "공유" 클릭
  2. "일반 액세스" → "링크가 있는 모든 사용자"
  3. 권한: "뷰어"

### 독립 실행 시 (standalone.html)
- CORS 프록시를 사용하여 데이터를 가져옵니다
- 3개의 프록시를 자동으로 시도 (allorigins.win, corsproxy.io, cors-anywhere)
- 프록시 실패 시 GitHub Pages나 Google Apps Script 방법 사용 권장

### 최소 데이터 요구사항
- **일별 대시보드**: 최소 1일의 데이터
- **전주 대비 증감률**: 최소 14일의 데이터 (최근 7일 + 이전 7일)

### 브라우저 호환성
- Chrome, Firefox, Safari, Edge 최신 버전 지원
- IE는 지원하지 않음

## 📈 확장성

### ✅ 자동 확장 기능
- **지점 추가**: 구글 시트에 새 지점 데이터만 입력하면 자동으로 대시보드에 표시
- **키워드 추가**: 새 키워드도 자동으로 인식하여 표시
- **날짜 추가**: 매일 데이터를 추가하면 날짜 드롭다운에 자동으로 추가
- **코드 수정 불필요**: 구글 시트만 업데이트하면 됨!

### 📊 지원 규모
- **현재 (7개 지점)**: 완벽하게 동작 ✅
- **소규모 (1~20개 지점)**: 최적화 불필요, 매우 빠름
- **중규모 (20~50개 지점)**: 필터 기능 추가 권장
- **대규모 (50개 이상)**: 성능 최적화 필요
